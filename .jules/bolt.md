
## 2026-03-01 - Optimizing React List Rendering with React.memo and useCallback
**Learning:** Wrapping a list item component like `EventCard` in `React.memo` is only effective if the props passed from the parent are stable. If the parent component (e.g., `App`) creates new closure-based event handlers inline (`onClick={() => setSelectedEventId(id)}`), it defeats memoization and causes every list item to re-render whenever the parent state changes (such as selecting a new item or updating a filter).
**Action:** When optimizing list components with `React.memo`, ensure that callback props (`onClick`, `onDelete`, etc.) are wrapped in `useCallback` at the parent level and receive necessary data (like `id` or `EventData` objects) as arguments instead of relying on closures. Furthermore, when these callbacks update parent state, use functional state updates (`prev => ...`) to avoid depending on the actual state object, keeping the callback reference perfectly stable across all renders.
