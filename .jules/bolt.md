## 2026-02-28 - React Memoization and State Update Side-effects
**Learning:** React strict mode may invoke functional state updates twice to help find accidental impurities. Placing side-effects (like updating a separate history state array) inside a functional state update `setEvents(prev => { ... setHistory(...); return next; })` is an anti-pattern and can lead to unexpected duplicate side-effects. While it removes the need for `events` in the dependency array (thereby stabilizing the function reference for child memoization), a safer pattern is to use a `useRef` to track the latest state without triggering re-renders, allowing the handler to remain stable without side-effects in the updater.
**Action:** Always avoid side-effects in React functional state updates. When stable function references are required that also depend on the latest state, use a `useRef` to store the latest state value and access it within the `useCallback` instead of using the state updater for side-effects.
